# AIEQ

[Английская версия (English version)](README.md)

AIEQ - параметрический эквалайзер для Windows 10/11 с ручным редактированием фильтров, сравнением пресетов, плагином автонастройки и ИИ чат-ботом. Приложение рассчитано на работу через виртуальный аудиокабель VB-Cable: звук из системы отправляется во вход приложению AIEQ, обрабатывается фильтрами и возвращается на выбранное выходное устройство.

![Скриншот: общий вид приложения](screenshots/1.png)

## 1. Возможности

- график АЧХ с текущим пресетом, сохраненными пресетами для сравнения и АЧХ выбранного устройства;
- фильтры `колокол`, `нижняя полка`, `верхняя полка`, `НЧ-фильтр`, `ВЧ-фильтр`, `НВЧ-фильтр`, `вырез`;
- огибающий режим сведения полос: пересекающиеся фильтры не суммируются каскадом;
- импорт/экспорт JSON-пресетов и SQLite-хранилище пресетов в `%APPDATA%\AIEQ`;
- ИИ-чат через локальную GGUF-модель и `llama-cpp-python`;
- поддержка официального backend пакета [`autoeq`](https://github.com/jaakkopasanen/AutoEq) на Python 3.11 с почти 7000 кривыми различных устройств на выбор.
- плагин автонастройки с 2 решениями: оригинальный алгоритм от разрабочтика выше упомянутого пакета (`jaakkopasanen`) и его доработанная версия от меня (`dmitryz1024`).

[Скринкаст](https://drive.google.com/file/d/1CIRGF4145KUJEnWvs8e4yJUtxBlE__0m/view?usp=sharing)

## 2. Установка и запуск

### Через setup.exe

1. Скачайте установщик [`aieq-<version>-setup.exe`](https://github.com/dmitryz1024/aieq/releases/tag/v0.1.0) из GitHub Releases.
2. Запустите установщик от имени обычного пользователя. Если Windows запросит права администратора, подтвердите.
3. Выберите папку установки. Внутри нее будет создана папка `aieq`.
4. Дождитесь, пока установщик скачает стандартную ИИ-модель в папку `models`.
5. Если установщик предложит установить VB-Cable, выполните установку драйвера.
6. После установки VB-Cable желательно перезагрузить компьютер.

После установки рядом с `AIEQ.exe` должны быть папки:

```text
assets\
curves\
languages\
models\
runtime\
```

### Через клонирование репозитория

Клонируйте репозиторий:

```powershell
git clone https://github.com/dmitryz1024/aieq
```

Запустите команду:

```powershell
uv sync --python 3.11 --extra dev --extra autoeq --extra ai
uv run python -m source
```

ИИ модель в формате .gguf ищите здесь: [тык](https://huggingface.co/), помещайте в папку `models`.

Необходимые для работы модели на графическом процессоре .dll и .exe файлы ищите здесь: [тык](https://github.com/ggml-org/llama.cpp/releases), помещайте в папку `runtime`.

Запуск тестов (pytest, ruff, pyrefly) осуществляется следующей командой:
```powershell
uv run python -m tests.check_all
```

## 3. Настройка аудиомаршрута

Типовой маршрут:

1. В Windows выберите `CABLE Input` как устройство вывода по умолчанию.
2. В AIEQ в поле `Вход` выберите устройство, соответствующее VB-Cable output.
3. В поле `Выход` выберите реальные наушники, колонки или внешний ЦАП.
4. Выберите частоту дискретизации (`ЧД (Гц)`) и формат выходного потока. Если сомневаетесь, оставьте вариант по умолчанию.
5. Нажмите кнопку запуска.

![Скриншот: выбор входа и выхода](screenshots/2.png)

Если звук не идет:

- в первую очередь убедитесь, что используете одинаковую подсистему и для входа, и для выхода (указывается в квадратных скобках перед названием устройства)
- нажмите кнопку обновления списков аудиоустройств;
- проверьте, что VB-Cable установлен и доступен в настройках Windows;
- убедитесь, что в Windows звук действительно отправляется в `CABLE Input`;
- попробуйте разные подсистемы для запуска потока, если доступно несколько.

## 4. Работа с графиком

В секции АЧХ отображаются:

- кривая выбранного устройства;
- текущий редактируемый пресет;
- целевая кривая автонастройки, если она включена;
- сохраненные пресеты для сравнения.

Кнопка `Сравнить` открывает список сохраненных пресетов. Можно отметить несколько вариантов или выбрать `Как можно больше`, если нужно быстро показать доступные пресеты в пределах лимита отображения (15 штук).

![Скриншот: график и сравнение пресетов](screenshots/3.png)

## 5. Устройства и целевые кривые

Кривые устройств лежат в:

```text
curves\devices\
```

Целевые кривые лежат в:

```text
curves\targets\
```

Чтобы добавить новые кривые:

1. Скопируйте `.txt` или `.csv` файлы в нужную папку.
2. В приложении нажмите кнопку обновления списков кривых.
3. Выберите новую кривую в списке.

Формат кривой: частота и значение в дБ в двух колонках, например:

```text
20 -3.1
100 -1.4
1000 0.0
10000 2.2
```

## 6. Ручное редактирование фильтров

Один фильтр содержит:

- `Тип` - тип фильтра;
- `Усиление` - изменение уровня в дБ;
- `Частота` - центральная частота или частота среза;
- `Q-фактор` - добротность, влияющая на ширину воздействия.

Доступные типы:

- колокол;
- нижняя полка;
- верхняя полка;
- НЧ-фильтр;
- ВЧ-фильтр;
- НВЧ-фильтр;
- вырез.

Более подробное описание каждого типа доступно в самом приложении при нажатии на мини-иконку в карточке фильтра, слева от названия типа.

![Скриншот: карточки фильтров](screenshots/4.png)

Для большинства музыкальных задач лучше начинать с `колокол`, `нижняя полка` и `верхняя полка`. Фильтры `НЧ-фильтр`, `ВЧ-фильтр`, `НВЧ-фильтр` и `вырез` лучше использовать осторожно: они сильнее меняют структуру сигнала и чаще нужны для специальных задач.

## 7. Пресеты

Текущий пресет выбирается в списке `Текущий пресет`. Пункт `Новый` создает новый редактируемый пресет.

Доступные действия:

- сохранить пресет;
- удалить пресет;
- импортировать в формате JSON;
- экспортировать в формате JSON;
- сравнить текущий пресет с сохраненными.

## 8. Автонастройка

Вкладка автонастройки строит пресет по выбранной кривой устройства и целевой кривой (подстраивает кривую текущего устройства под целевую).

1. Выберите устройство.
2. Откройте вкладку автонастройки.
3. Выберите целевую кривую.
4. Выберите алгоритм.
5. Нажмите `Рассчитать автонастройку`.

После расчета новый пресет сохраняется, становится текущим и отображается на графике.

![Скриншот: вкладка автонастройки](screenshots/5.png)

## 9. ИИ чат-бот

ИИ чат-бот принимает обычный текстовый запрос и возвращает новый пресет. Модель получает контекст текущего устройства, текущего пресета, истории чата и, если запрос содержит имя какого-либо из уже сохраненных пресетов, данные этого пресета.

Примеры запросов:

```text
Сделай звук мягче и спокойнее, без резкой верхней середины.
```

```text
Добавь немного плотности басу, но не трогай саббас.
```

```text
Доработай пресет AIEQ 2026-05-06 | 01-59-52: слегка подними вокал.
```

![Скриншот: AI-чат](screenshots/6.png)

Если модель не подключена, приложение покажет системное уведомление о том, что ИИ-агент не подключен.

## 10. Модели ИИ

Модели лежат в:

```text
models\
```

Чтобы добавить модель:

1. Скопируйте `.gguf` файл в `models`.
2. Нажмите кнопку обновления списка моделей.
3. Выберите модель в выпадающем списке.

Для работы ИИ модели на графическом процессоре `llama-server.exe` и DLL должны лежать в:

```text
runtime\
```

Проверка доступности графического процессора:

```powershell
.\runtime\llama-server.exe --list-devices
```

В выводе должен быть `CUDA0`.

## 11. Языки интерфейса

Языковые файлы лежат в:

```text
languages\
```

Чтобы добавить язык:

1. Создайте новый `.json` по образцу `ru.json` или `en.json`.
2. Положите файл в папку `languages`.
3. Откройте настройки приложения.
4. Нажмите обновление списка языков.
5. Выберите новый язык.

## 12. Замена изображений интерфейса

Изображения лежат в:

```text
assets\
```

Их можно заменить вручную. Изменения подтянутся после перезапуска приложения.

## 13. Настройки

Кнопка настроек открывает параметры приложения. Там можно:

- посмотреть текущую задержку аудиопотока;
- включить пользовательскую задержку;
- включить расширенные настройки ИИ;
- разрешить или запретить использование центрального процессора для просчета ответов ИИ;
- сменить язык интерфейса.

![Скриншот: окно настроек](screenshots/7.png)

## 14. Полезные скрипты

В папке `scripts` вы найдете ряд полезных подпрограмм, позволяющих:
- сбросить размеры оконного режима до значений по умолчанию
```powershell
uv run python -m scripts.reset_window_layout
```
- синхронизировать списки кривых устройств с последними обновлениями пакета [`autoeq`](https://github.com/jaakkopasanen/AutoEq)
```powershell
uv run python -m scripts.sync_autoeq_curves
```
- проверить поддерживаемые параметры потока конкретной пары устройств входа/выхода
```powershell
uv run python -m scripts.check_audio_formats
```

## 15. Частые проблемы

### Нет звука

- Проверьте все этапы формирования звукового потока: Windows -> VB-Cable -> AIEQ -> устройство вывода.
- Нажмите обновление аудиоустройств.
- Попробуйте другую подсистему того же устройства.
- Убедитесь, что поток приложения запущен.

### ИИ не отвечает

- Проверьте, что модель лежит в `models`.
- Проверьте, что `runtime\llama-server.exe` существует.
- Выполните `runtime\llama-server.exe --list-devices`.
- Если графический процессор недоступен, разрешите использование центрального в настройках (только для диагностики).

### Автонастройка долго считает

Оригинальный алгоритм автонастройки может быть чуть медленнее локального. Попробуйте воспользоваться обоими алгоритмами по очереди, чтобы выявить наиболее оптимальный для ваших задач.

### Новые файлы не появились в списках

Нажмите соответствующую кнопку обновления:

- языки - в настройках;
- модели - во вкладке ИИ;
- кривые - в секции АЧХ;
- аудиоустройства - в верхней строке маршрута.
